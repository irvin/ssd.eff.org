如何：在 Windows 上安全地刪除您的資料

下載位置：http://bleachbit.sourceforge.net/download/windows

電腦需求：Windows XP 或更新版本

本指引中使用的版本：BleachBit 1.6

授權：GPL

級別：初學者

所需時間：10 分鐘到幾個小時 (取決於要安全刪除的檔案/磁碟大小)

我們大多認為，一旦把檔案放進垃圾桶並清空，電腦上的檔案就會被刪除；其實，刪除檔案並不會完全將其抹除。這麼做的時候，電腦只是讓使用者看不到檔案，並將檔案存放在磁碟上的部分表示為「可用」- 表示您的作業系統現在可以用新資料覆寫檔案。因此，可能需要幾週、幾個月甚至幾年，才會被新的檔案覆寫。在這種情況發生前，該「已刪除」檔案仍在您的磁碟上，只是對一般操作是不可見的。藉由一點工夫和正確的工具 (如「反刪除」軟體或鑑識方法)，您甚至可以取回「已刪除」的檔案。重點在於電腦通常不「刪除」檔案，只是允許這些檔案佔用的空間將來會被別的東西覆寫。

那麼永遠刪除一個檔案的最好辦法，就是確保它被立即覆寫，這樣就很難取回原本寫在那裡的東西。您的作業系統可能已經有能這麼做的軟體 - 可以用廢資料將磁碟上所有「空」空間覆寫的軟體，保護已刪除資料的機密性。

請注意，安全刪除固態硬碟 (SSD)、USB 隨身碟和 SD 卡中的資料非常困難！下面的說明僅適用於傳統的磁碟，而不適用於在現代筆記型電腦成為標準配備的 SSD、USB key/USB 隨身碟或 SD 卡/記憶卡。

這是因為這些類型或磁碟機使用稱為耗損平均技術的技術。(您可以在這裡了解更多關於為什麼這會導致安全刪除的問題。)

如果您使用 SSD 或 USB 隨身碟，則可以跳到下面的部分。

在 Windows 上，我們目前建議使用 BleachBit。BleachBit 是一個用於 Windows 和 Linux 的免費/開源安全刪除工具，比內建的 Cipher.exe 複雜得多。

可以使用 BleachBit 快速輕鬆找出單一檔案進行安全刪除，或者實施定期的安全刪除原則。也可以自行編寫檔案刪除指令。請查看此文件以獲得更多資訊。

取得 BleachBit Anchor link

您可以從 BleachBit 下載頁面下載安裝程式，取得 Windows 版 BleachBit

按一下 BleachBit 安裝程式的 .exe 連結。您會前往下載頁面。

許多瀏覽器會要求您確認是否要下載此檔案。Internet Explorer 11 會在瀏覽器視窗底部顯示一個帶有橘色邊框的欄。

對於任何瀏覽器，最好在繼續之前先儲存檔案，所以按一下「儲存」(Save) 按鈕。預設情況下，大多數瀏覽器將下載的檔案存放在 Downloads 檔案夾。

安裝 BleachBit Anchor link

開著 Windows 檔案總管視窗，並點兩下 BleachBit-1.6-setup。系統會詢問您是否要允許安裝這個程式。按一下「是」(Yes) 按鈕。

畫面上將打開一個視窗，要求您選擇一種安裝語言。選擇您想要的語言，然後按一下確定 (OK) 按鈕。

下一個視窗將顯示 GNU 通用公共授權條款。選取接受方塊，然後按一下下一步 (Next) 按鈕。

在下一個視窗中，BleachBit 會顯示一些自訂選項。您可以保留原樣不更動。我們建議取消勾選桌面 (Desktop) 選項。按一下下一步 (Next) 按鈕。

現在 BleachBit 將再次要求您確認是否要安裝。按一下安裝 (Install) 按鈕。

最後，BleachBit 安裝程式會顯示一個視窗，告訴您安裝完成。按一下下一步 (Next) 按鈕。

安裝程式的最後一個視窗會詢問您是否要執行 BleachBit。取消勾選執行 BleachBit (Run BleachBit) 的選項。按一下完成 (Finish) 按鈕。

使用 BleachBit Anchor link

BleachBit 介面

前往開始功能表，按一下 Windows 圖示，然後從功能表中選取 BleachBit。

畫面上會打開一個小視窗，確認您想開啟 BleachBit。

BleachBit 主視窗將開啟。BleachBit 將檢測幾個已安裝的常用程式，並顯示每個程式的特殊選項。BleachBit 有四個預設設定。

使用預設值

BleachBit 可以使用 Internet Explorer 預設值抹除 Internet Explorer 留下的記錄 (但是 BleachBit 無法抹除其他瀏覽器的記錄)。核取 Internet Explorer 旁邊的方塊。注意所有屬於 Cookies、表單歷程 (Form history)、瀏覽歷程 (History) 和暫存檔案 (Temporary files) 選項的方塊也要選取。您可以根據需要取消勾選。按一下清理 (Clean) 按鈕。

BleachBit 現在將清理某些檔案並顯示進度。

安全地刪除檔案夾

按一下檔案 (File) 功能表，然後選擇粉碎檔案夾 (Shred Folders)。

畫面上會打開一個小視窗。選擇您想要粉碎的檔案夾。

BleachBit 將要求您確認是否要永久刪除您選擇的檔案。按一下刪除 (Delete) 按鈕。

BleachBit 現在將顯示您刪除的檔案。請注意，BleachBit 會安全地刪除檔案夾中的每個檔案，然後安全地刪除該檔案夾。

安全地刪除檔案

按一下檔案 (File) 功能表，然後選擇粉碎檔案 (Shred Files)。

檔案選擇視窗將打開。選擇您想要粉碎的檔案。

BleachBit 將要求您確認是否要永久刪除您選擇的檔案。按一下刪除 (Delete) 按鈕。

BleachBit 有一些其他功能。最有用的可能是「抹除可用空間」(wipe free space)。這將嘗試抹除您已經刪除的任何檔案的記錄。通常 Windows 會將已刪除檔案的全部或部分資料留在剩餘的可用空間中。「抹除可用空間」會將隨機資料覆寫硬碟的這些空白部分。抹除可用空間可能需要很長的時間，具體取決於您的磁碟有多少剩餘容量。

關於安全刪除工具之限制的警告 Anchor link

首先，請記住，上述建議只會刪除您正在使用的電腦硬碟上的檔案。上述任何工具都不會刪除在您的電腦其他地方、另一個磁碟或 USB 隨身碟、「Time Machine」、電子郵件服務器或雲端中所做的備份。要安全地刪除檔案，必須刪除該檔案的每一個副本，無論它存放或發送到何處。此外，一旦檔案存放在雲端 (例如藉由 Dropbox 或其他檔案共享服務)，那麼通常無法保證永久刪除。

不幸的是，安全刪除工具還有另一個限制。即使您按照上述建議刪除了檔案的所有副本，也可能會在電腦上保留某些已刪除檔案的痕跡，並不是因為檔案本身沒有被妥善刪除，而是因為部分作業系統或者其他程式刻意保留的記錄。

這種情況發生的方式有很多，但有兩個例子足以說明這種可能性。在 Windows 或 Mac OS 上，即使檔案已被刪除，Microsoft Office 也可能保留對「最近檔案」功能表中檔案名稱的參照 (Office 有時甚至可能保留包含檔案內容的暫存檔)。在 Linux 或其他 *nix 系統上，OpenOffice 可能會保留與 Microsoft Office 一樣多的記錄，而且使用者的 shell 歷史記錄檔可能含有包含檔案名稱的指令，即使檔案已被安全刪除。在實務中，可能會有幾十個這樣做的程式。

很難知道如何因應這個問題。可以假設即使一個檔案已被安全刪除，它的名稱仍可能在電腦上持續存在一段時間。覆寫整個磁碟是 100% 確定名稱不復存在的唯一方法。

丟棄舊硬體時安全刪除資料 Anchor link

如果您終於想扔掉一件硬體或者在 eBay 上賣掉，您會想確保沒有人能夠從中獲取您的資料。研究一再發現，電腦所有者通常無法做到這一點 - 硬碟轉售時，經常充滿高度敏感的資訊。因此，在銷售或回收電腦之前，請務必先用廢資料覆寫儲存媒體。即使您沒有立即扔掉它，如果您的電腦已經達到了使用壽命而不再使用，在把電腦存放在角落或櫃子裡之前，抹除硬碟也比較安全。Darik's Boot and Nuke 就是為此目的而設計的工具，而且網路上有關於如何使用的各種教學 (包括這裡)。

一些全機加密軟體有能力銷毀主要金鑰，讓硬碟的加密內容永久無法解讀。由於金鑰資料很少，幾乎可以立即破壞，所以比像是 Darik's Boot and Nuke 這樣的覆寫軟體速度要快得多，後者對於更大的磁碟來說可能相當耗時。但是，只有硬碟始終加密，這個選項才是可行的。如果您沒有提前使用全機加密，則需要在丟棄之前覆寫整個磁碟。

丟棄 CD-ROM Anchor link

對於 CD-ROM，您應該把它當廢紙一樣看待，將它們碎掉。有價格實惠的碎紙機能絞碎 CD-ROM。不要把 CD-ROM 扔進垃圾桶，除非您確定裡面沒有任何敏感的東西。

安全刪除固態硬碟 (SSD)、USB 隨身碟和 SD 卡中的資料 Anchor link

不幸的是，由於 SSD、USB 隨身碟和 SD 卡的運作方式，安全刪除單個檔案和空閒空間即使並非不可能，也非常困難。因此，就保護而言，最好的辦法就是使用加密，這樣的話，即使檔案仍在磁碟上，對於任何取得磁碟的人來說，至少看起來都像是廢資料，而且無法強迫您將它解密。此時我們還無法提供一個良好的程序，必定從 SSD 中刪除您的資料。如果您想知道資料為什麼這麼難刪除，請繼續閱讀。

正如我們上面提到的，SSD 和 USB 隨身碟使用稱為耗損平均的技術。在高階運作上，耗損平均技術運作方式如下。每個磁碟上的空間被分成區塊，就像書頁一樣。一個檔案寫入磁碟時，它被分配給某個區塊或一組區塊 (頁面)。如果您想覆寫檔案，那麼就要告訴磁碟覆寫這些區塊。但是在 SSD 和 USB 隨身碟中，抹除和反覆寫入同一個區塊會造成耗損。每個區塊只能抹除和反覆寫入有限次數，之後該區塊就不會再運作 (就跟紙張一樣，如果您用鉛筆反覆寫字和擦除，最後紙可能會破掉不堪使用)。為解決這個問題，SSD 和 USB 隨身碟會試圖確保每個區塊被抹除和反覆寫入的次數大致相同，這樣磁碟就能盡可能延長壽命 (因此出現「耗損平均」這個術語)。這個技術的副作用是，有時磁碟不會抹除和寫入檔案最初存放的區塊，而是將該區塊單獨留下，將其標示為無效，並且將修改的檔案寫入不同的區塊。這就好像將書頁維持不變，將修改後的檔案寫在不同的頁面，然後更新書本的目錄，指向新的頁面。所有這一切都發生在磁碟非常低階的電子活動上，所以作業系統甚至沒有意識到這件事。這表示即使您嘗試覆寫檔案，也不能保證磁碟會覆寫檔案 - 這就是為什麼使用 SSD 進行安全刪除要困難得多。
