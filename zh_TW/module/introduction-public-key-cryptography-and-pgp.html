公開金鑰加密和 PGP 簡介

PGP 代表優良保密協定 (Pretty Good Privacy)。這確實是非常好的保密協定。如果使用得當，即使資金充足的政府監控計劃也不能存取您的訊息、文字甚至檔案的內容。Edward Snowden 說「加密真的有用」，他說的正是 PGP 及其相關軟體。應該指出的是，政府竊取私人電腦的私密金鑰並不是前所未聞的 (藉由將電腦帶走，或將惡意程式透過實體存取或網路釣魚攻擊植入)，這麼做能解除保護，甚至可以閱讀舊郵件。這相當於您的門上可能有一個無法撬開的鎖，但有人仍然可以在街上扒走您的金鑰，然後複製它，再放回您的口袋，因此進入您的房子，甚至不用撬開鎖。

不幸的是，PGP 在理解難度或使用方面也表現相當不好。PGP 使用的強式加密 - 公開金鑰加密 - 很巧妙，但很難讓您理解。PGP 軟體本身自 1991 年以來一直存在，這讓它與 Microsoft Windows 的早期版本一樣復古，從那時起它的外觀就沒有太大變化。

好訊息是，現在有很多可用的程式可以隱藏古老的 PGP 設計，並且使用起來更容易一些，特別是對電子郵件進行加密和認證時，這也是 PGP 的主要用途。我們已經納入了在其他地方安裝和操作這個軟體的指引。

使用 PGP 或其他使用它的程式之前，花點時間去理解公開金鑰加密的基本知識：它可以為您做什麼、不能做什麼，以及什麼時候該使用它。

兩把鑰匙的故事 Anchor link

使用加密對抗監控時，這是我們嘗試做的事情：

我們拿一則清晰可讀的訊息，如「嗨，媽媽」。我們把它加密編成一則難以理解的編碼訊息 (例如「OhsieW5ge+osh1aehah6」)。我們藉由網際網路發送加密的訊息，在那裡可以被很多人閱讀，但希望沒有任何人讀得懂。然後，等它到達目的地，我們的預期收件人，而且只有我們預期的收件人，有一些方法將其解密回原始資訊。

我們的收件人怎麼知道如何解碼訊息，而其他人不知道？這一定是因為他們知道一些別人不知道的額外資訊。我們稱之為解碼金鑰，因為它解鎖了代碼中的訊息。

收件人如何知道這組金鑰？主要是因為寄件人先前已經告訴他們鑰匙，不管是「試著把訊息照鏡子」，還是「把每個字母轉換成字母表中的下一個字母」。然而這個策略有一個問題。如果您擔心在發送編碼訊息時受到監視，要如何向接收方發送金鑰，而不會讓那個談話也遭到監聽？如果您的攻擊者已經知道解碼金鑰，那麼發送巧妙加密的訊息也毫無意義。如果您有祕密方法能發送解碼金鑰，為什麼不把它用於所有的祕密訊息呢？

公開金鑰密碼學有一個很棒的解決方案。談話中的每個人都有建立兩組金鑰的方法。一組是他們自己保存的私密金鑰，永遠不會讓別人知道。另一組是公開金鑰，發送給任何想與他們通訊的人。誰能看到公開金鑰並不重要。您可以把它放在網路上，每個人都可以看到它。

這些「鑰匙」本身其實是非常大的數字，具有一定的數學性質。公開金鑰和私密金鑰是相關的。如果使用公開金鑰對某些內容進行編碼，則其他人可以使用其相符的私密金鑰對其進行解碼。

我們來看看那是怎麼辦到的。您想發送一個祕密訊息給 Aarav。Aarav 有一組私密金鑰，但就像其他優秀的公開金鑰加密使用者，他把跟私密金鑰相關的公開金鑰放在他的網頁上。您下載公開金鑰，使用它加密訊息，並將訊息發送給他。他可以進行解碼，因為他有對應的私密金鑰，但沒有其他人可以。

時代象徵 Anchor link

公開金鑰密碼學擺脫了得將解密金鑰偷偷拿給您的訊息接收對象的問題，因為那個人已經有了金鑰。您只需要掌握相符的公開加密金鑰，接收方可以發送給任何人，包括間諜。因為它僅用於編碼訊息，所以對於任何試圖解碼訊息的人都沒有用。

但是等等！如果使用特定的公開金鑰對訊息進行編碼，則只能藉由相符的私密金鑰進行解碼。反之亦然。如果您使用某個私密金鑰對訊息進行編碼，則只能使用與其相符的公開金鑰進行解碼。

為什麼這會有用？乍看之下，用私密金鑰祕密發送資訊似乎沒有什麼好處，世界上每個人 (或者至少每個有公開金鑰的人) 都可以破解。但是，假設我寫了一則訊息說：「我答應支付 Aazul $100」，然後用我的私密金鑰把它變成祕密訊息，任何人都可以解碼這個訊息 - 但是只可能是一個人寫下這個訊息：擁有我私密金鑰的人。如果我把我的私密金鑰安全地保存好了，那就表示是我，而且只有我。實際上，藉由使用我的私密金鑰進行編碼，我就確保這個訊息只能是由我發出。換句話說，就像我們在現實世界中簽名的時候一樣，我對這個數位訊息也做了同樣的事情。

簽名也能防止訊息遭到篡改。如果有人試圖將「我答應支付 Aazul $100」改為「我答應支付 Bob $100」，他們將無法使用我的私密金鑰重新簽名。所以簽名的訊息可保證來自於某個來源，而不會在傳輸過程中遭到竄改。

所以公開金鑰密碼學可讓您加密並將訊息安全發送給您知道公開金鑰的任何人。如果其他人知道您的公開金鑰，他們可以發送訊息給您，只有您可以解碼。而且如果他人知道您的公開金鑰，您可以為訊息簽名，以便這些人知道訊息只有可能從您那裡來。如果您知道別人的公開金鑰，您可以解碼他們簽名的訊息，並知道只有可能來自他們。

現在應該已經很清楚，公開金鑰密碼學變得越來越有用，越來越多的人會知道您的公開金鑰。同樣明顯的是，您需要將您的私密金鑰保護得非常好。如果其他人得到您私密金鑰的副本，他們可以冒充您，並為訊息簽名，聲稱是您寫的。PGP 有一個功能，可讓您「撤銷」私密金鑰，並警告他人金鑰已不再可信，但這不是一個很好的解決方案。使用公開金鑰密碼學系統最重要的是非常小心地保護您的私密金鑰。

PGP 如何運作 Anchor link

優良保密協定主要關注建立、使用公開金鑰及私密金鑰的細節。您可以用它建立一個公開金鑰/私密金鑰對，以密碼保護私密金鑰，並使用它和您的公開金鑰進行簽名及加密文字。它也可讓您下載他人的公開金鑰，並將您的公開金鑰上傳到「公開金鑰伺服器」，這是其他人可找到您金鑰的存放庫。請參閱我們的指引，在您的電子郵件軟體中安裝與 PGP 相容的軟體。

如果您要從這個概述中學到一件事，那就是：您應該將私密金鑰存放在安全的地方，並使用長密碼保護。您可以把您的公開金鑰給任何您想通訊的對象，或者想知道一個訊息是否真正由您發出的人。

先進的 PGP：信任網 Anchor link

您可能已經發現了公開金鑰密碼學的潛在缺陷。假設我開始發布一個我說來自歐巴馬的公開金鑰。如果他人相信我，他們可能會開始向歐巴馬發送祕密訊息，使用這組金鑰加密。或者他們可能認為用這個金鑰簽名的任何事情都是來自歐巴馬的保證。這是非常罕見的，但卻真實發生在一些現實生活中的人身上，包括本文的一些作者 - 有些寫信給他們的人被愚弄了！(在這些情況下，我們並不知道某些製作假金鑰的人是否真的能夠攔截並讀取傳送中的資訊，或者是否只是種惡作劇，造成加密交談時的不便。)

另一種暗中攻擊，是攻擊者坐在兩個線上交談的人之間，竊聽他們的全程對話，偶爾把攻擊者自己的誤導訊息插入對話中。由於網際網路的設計是一個藉由許多不同的電腦與私人方傳送資訊的系統，這種攻擊是完全可能的。在這種情況下 (稱為「攔截式攻擊」)，沒有事先協議就交換金鑰，可能會非常危險。「這是我的金鑰」，一個聲稱是歐巴馬的人，向您發送一個公開金鑰檔案。但是如果有人不等到那個時候，攔截歐巴馬的金鑰傳送，然後換成自己的金鑰呢？

我們如何證明某組金鑰確實屬於某個人？一種方法是直接從他們身上獲得金鑰，但這並不比我們最初的挑戰好到哪裡去，也就是必須私下取得金鑰。但是，目前還是有人會在私人和公開加密聚會 (cryptoparty) 時交換公開金鑰。

PGP 有一個更好的解決方案，稱為「信任網」。在信任網中，如果我相信某個金鑰屬於某個人，那麼我可以對該金鑰簽名，然後將金鑰 (和簽名) 上傳到公開金鑰伺服器。然後這些金鑰伺服器將把簽名的金鑰傳給任何向其索取的人。

簡單一點說，越多我相信的人簽了一組金鑰，我就越有可能相信這把鑰匙真的屬於其聲稱的對象。PGP 可以讓您簽署其他人的金鑰，也可以讓您信任其他簽名者，所以如果他們簽了一個金鑰，您的軟體會自動相信該金鑰是有效的。

信任網有自己的挑戰，像 EFF 這樣的組織正在研究更好的解決方案。但現在如果您想要一個將金鑰親自交給他人的替代方案，那麼使用信任網和公開金鑰伺服器網路是您的最佳選擇。

中繼資料：PGP 的限制 Anchor link

PGP 主要確保資訊內容是祕密、真實、不受竄改的。但這不是您可能擁有的唯一隱私問題。正如我們已經提到的那樣，關於您訊息的資訊可以像其內容一樣揭露許多事情 (請參閱「中繼資料」)。如果您正在與您所在國家的異議分子交換 PGP 訊息，即使這些訊息沒有解碼，只要您與其通訊，您也可能處於危險之中。而在一些國家，您確實可能因為拒絕將加密訊息解密而遭到監禁。

PGP 不會掩飾您正在與誰交談，或者您正在使用 PGP 這麼做。如果您上傳您的公開金鑰到金鑰伺服器上，或簽署其他人的鑰匙，您就向全世界說明了您的金鑰是什麼，以及您認識誰。

您可以不必這樣做。您可以保持您的 PGP 公開金鑰低調，只給您感到安全的人，並告訴他們不要上傳到公開金鑰伺服器。您不需要將您的名字附加到一組金鑰上。

要掩蓋您在與一個特定的人溝通是比較困難的。一種方法是你們都使用匿名電子郵件帳戶，並使用 Tor 存取它們。如果您這樣做，PGP 將仍然是有用的，既保持您電子郵件訊息的私密性，並能向彼此證明，訊息沒有遭到篡改。
